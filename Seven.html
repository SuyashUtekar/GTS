BERTopic Embedding Model
Evaluation on Indian Legal Docs

A systematic pipeline to identify the best embedding model for legal topic clustering

Phase 01
1
📂 Document Collection & Preparation

Gather 10–20 Indian legal documents spanning diverse case types. Ensure variety to stress-test topic separation across embedding models.

Supreme Court judgments High Court orders IPC sections PIL documents Contract disputes
🧹 Preprocessing Pipeline

Clean and normalize each document before embedding. Use a shared preprocessing function so all models receive identical input.

Remove headers/footers Strip citations noise Sentence tokenization Min length filter
Phase 02
2
🧠 Embedding Model Selection

Test 3–5 embedding models. Cover general-purpose, legal-domain, and multilingual options to find what works best on Indian legal language.

all-MiniLM-L6-v2
Fast, general-purpose. Good baseline.
legal-bert-base
Fine-tuned on legal corpora. Domain-aware.
paraphrase-mpnet-base-v2
Strong semantic similarity.
multilingual-e5-large
Handles Hindi/regional legal terms.
bge-large-en-v1.5
SOTA retrieval embeddings.
Phase 03
3
⚙️ BERTopic Configuration (Shared)

Use identical BERTopic hyperparameters across all models so only the embedding changes between runs. This ensures a fair apples-to-apples comparison.

UMAP: n_neighbors=15, n_components=5 HDBSCAN: min_cluster_size=5 CountVectorizer: stop_words=english top_n_words=10

Decision Criteria

Rank models using a weighted score: Topic Coherence (40%) + Silhouette Score (30%) + Topic Diversity (20%) + Manual Legal Relevance (10%). The model with the highest weighted score AND interpretable legal topics wins. If two models tie on metrics, prefer the one with fewer outlier documents.
